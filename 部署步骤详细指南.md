# 🚀 网站部署详细步骤指南

## 第一步：Git 仓库已初始化 ✅

我已经帮你初始化了 Git 仓库并创建了第一次提交。

---

## 第二步：创建 GitHub 仓库

### 2.1 登录 GitHub
1. 访问 https://github.com
2. 登录你的账号（如果没有账号，先注册）

### 2.2 创建新仓库
1. 点击右上角的 **"+"** 按钮
2. 选择 **"New repository"**
3. 填写仓库信息：
   - **Repository name**: `hover-web` （或你喜欢的名字）
   - **Description**: `HOVER WEB - 新媒体艺术风格内容展示网站`
   - **Visibility**: 选择 **Public**（公开）或 **Private**（私有）
   - ⚠️ **不要**勾选 "Initialize this repository with a README"（我们已经有了）
4. 点击 **"Create repository"**

### 2.3 复制仓库地址
创建完成后，GitHub 会显示仓库地址，类似：
```
https://github.com/你的用户名/hover-web.git
```
**请复制这个地址，下一步会用到！**

---

## 第三步：将代码推送到 GitHub

### 3.1 添加远程仓库

在终端中运行（将 `你的用户名` 和 `仓库名` 替换为实际值）：

```bash
git remote add origin https://github.com/你的用户名/仓库名.git
```

### 3.2 重命名分支为 main

```bash
git branch -M main
```

### 3.3 推送到 GitHub

```bash
git push -u origin main
```

**注意**：第一次推送可能需要登录 GitHub，按照提示操作即可。

---

## 第四步：部署到 Vercel

### 4.1 访问 Vercel
1. 访问 https://vercel.com
2. 点击右上角 **"Sign Up"** 或 **"Log In"**
3. 选择 **"Continue with GitHub"**（使用 GitHub 账号登录）

### 4.2 导入项目
1. 登录后，点击 **"Add New..."** → **"Project"**
2. 在 "Import Git Repository" 中找到你的仓库（`hover-web`）
3. 点击 **"Import"**

### 4.3 配置项目
Vercel 会自动检测到这是 Next.js 项目，通常不需要修改设置：
- **Framework Preset**: Next.js ✅
- **Root Directory**: `./` ✅
- **Build Command**: `npm run build` ✅
- **Output Directory**: `.next` ✅

### 4.4 配置环境变量 ⚠️ 重要！

在部署之前，**必须**配置环境变量：

1. 在项目配置页面，找到 **"Environment Variables"** 部分
2. 点击 **"Add"** 添加以下变量：

   **变量 1：**
   - **Name**: `NOTION_API_TOKEN`
   - **Value**: `你的_notion_token`（从 .env.local 文件中复制）
   - **Environment**: 选择所有环境（Production, Preview, Development）

   **变量 2：**
   - **Name**: `NEXT_PUBLIC_SITE_URL`
   - **Value**: `https://你的项目名.vercel.app`（先填这个，部署后会更新）
   - **Environment**: 选择所有环境

3. 点击 **"Save"**

### 4.5 开始部署
1. 点击 **"Deploy"** 按钮
2. 等待部署完成（通常 2-5 分钟）

### 4.6 获取网站地址
部署完成后，你会看到：
- ✅ **Production URL**: `https://你的项目名.vercel.app`
- 这就是你的网站地址！

### 4.7 更新环境变量（重要）
部署完成后，需要更新 `NEXT_PUBLIC_SITE_URL`：

1. 进入项目 **Settings** → **Environment Variables**
2. 找到 `NEXT_PUBLIC_SITE_URL`
3. 点击编辑，将值改为你的实际域名（如 `https://你的项目名.vercel.app`）
4. 点击 **"Save"**
5. 返回 **Deployments**，点击最新的部署右侧的 **"..."** → **"Redeploy"**

---

## 第五步：测试网站

1. 访问你的网站地址（如 `https://你的项目名.vercel.app`）
2. 检查网站是否正常加载
3. 检查 Notion 内容是否正常显示

---

## 第六步：自定义域名（可选）

如果你想使用自己的域名：

1. 在 Vercel 项目设置中进入 **"Domains"**
2. 输入你的域名（如 `hover-web.com`）
3. 按照提示配置 DNS 记录
4. 等待 DNS 生效（通常几分钟到几小时）

## 常见问题

### Q: 推送代码时提示需要认证？
A: 使用 GitHub Personal Access Token：
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. 生成新 token，勾选 `repo` 权限
3. 推送时使用 token 作为密码

### Q: 部署后网站显示错误？
A: 检查：
1. 环境变量是否正确配置
2. Notion API Token 是否正确
3. 查看 Vercel 的部署日志（Deployments → 点击部署 → View Function Logs）

### Q: 如何更新网站内容？
A: 
1. 修改代码
2. `git add .`
3. `git commit -m "更新内容"`
4. `git push`
5. Vercel 会自动重新部署
