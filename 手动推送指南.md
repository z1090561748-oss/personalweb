# 手动推送代码到 GitHub 指南

由于网络连接问题，请按照以下步骤手动推送代码：

## 方法一：在终端手动推送（推荐）

### 步骤 1：打开 PowerShell 或 CMD

在项目目录 `g:\notion` 中打开终端。

### 步骤 2：运行推送命令

```bash
git push -u origin main
```

### 步骤 3：如果提示需要认证

#### 选项 A：使用 Personal Access Token（推荐）

1. **获取 Token**：
   - 访问：https://github.com/settings/tokens
   - 点击 "Generate new token" → "Generate new token (classic)"
   - Note: `Git Push`
   - 勾选 `repo` 权限
   - 点击 "Generate token"
   - **复制 token**（只显示一次！）

2. **推送时输入**：
   - 用户名：`z1090561748-oss`
   - 密码：**粘贴刚才复制的 token**（不是 GitHub 密码）

#### 选项 B：使用 GitHub CLI

```bash
# 安装 GitHub CLI（如果还没有）
winget install --id GitHub.cli

# 登录
gh auth login

# 然后推送
git push -u origin main
```

## 方法二：使用 GitHub Desktop（最简单）

1. **下载安装**：
   - 访问：https://desktop.github.com
   - 下载并安装 GitHub Desktop

2. **登录**：
   - 打开 GitHub Desktop
   - 使用你的 GitHub 账号登录

3. **添加仓库**：
   - 点击 "File" → "Add Local Repository"
   - 选择 `g:\notion` 文件夹
   - 点击 "Add"

4. **发布**：
   - 点击 "Publish repository" 按钮
   - 确认仓库名：`personalweb`
   - 选择 "Keep this code private"（如果需要）
   - 点击 "Publish repository"

---

## 方法三：使用 VSCode（如果你用 VSCode）

1. 在 VSCode 中打开 `g:\notion` 文件夹
2. 点击左侧的源代码管理图标（或按 `Ctrl+Shift+G`）
3. 点击 "..." → "Push"
4. 如果提示，输入 GitHub 用户名和 Personal Access Token

## 如果还是遇到问题

可能的原因和解决方案：

1. **网络问题**：
   - 检查网络连接
   - 尝试使用 VPN
   - 稍后重试

2. **防火墙阻止**：
   - 检查防火墙设置
   - 允许 Git 和 PowerShell 访问网络

3. **代理设置**：
   - 如果使用代理，需要配置 Git 代理：
   ```bash
   git config --global http.proxy http://代理地址:端口
   ```
